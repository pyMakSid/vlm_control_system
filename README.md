Система, которая по видеопотоку определяет команды управления БПЛА с помощью
маловесной VLM. Кадр (опционально предразмеченный YOLO) вместе с системным и
миссионным промптами уходит в VLM, та возвращает одну команду управления.

## Структура

```
.
├── main.py             # работа на видеопотоке
├── start_server.py     # запуск сервера инференса
├── benchmark.py        # работа на mp4 → JSON-отчёт
├── pipeline.py         # полный конвейер
├── evaluate.py         # оценка качества вывода по result.json (без разметки)
├── src/
│   ├── camera.py         # фоновый захват
│   ├── vlm_client.py     # клиент VLM (streaming, окно контекста, метрики)
│   ├── prompts.py        # системный промпт + миссии
│   ├── metrics.py        # FPS / TTFT / TPS
│   ├── detector.py       # YOLO-предразметка кадра (опционально)
│   ├── llama_launcher.py # запуск llama-server
│   └── rkllm_server.py   # Flask-сервер на Rockchip NPU
```

## Быстрый старт

### Вариант 1 — всё одной командой (полный конвейер)

```bash
uv sync

# сервер → benchmark → evaluate → demo; артефакты в папку прогона
python pipeline.py --model path_to_llm --mmproj path_to_mmproj\
    --video flight.mp4 --mission straight \
    --yolo --yolo-model path_to_yolo --out-dir runs/test1

# видеопоток с камеры
python pipeline.py --run main --source 0 --mission follow
```

В режиме benchmark на выходе создаётся **папка с результатами работы** (`--out-dir`, по
умолчанию `runs/run_<время>`) с:
- `result.json` — ответы модели по кадрам (логи),
- `metrics.json` — метрики качества,
- `demo.mp4` — видео с YOLO-рамками и подписями команд,
- `run.log` — общий лог всех этапов.

`pipeline.py` парсит серверные флаги (`--hf`/`-m`/`--mmproj`/`--port`/`--ngl`/
`--ctx`), сам поднимает и гасит сервер, подставляет `--url`. `--dry-run` покажет
команды без запуска, `--keep-server` оставит сервер жить.

### Вариант 2 — вручную, два терминала

```bash
# терминал 1 — сервер
python start_server.py --model path_to_llm --mmproj path_to_mmproj\

# терминал 2 — клиент
python main.py --source 0 --mission follow

# или
python benchmark.py --video frames_dir/ --sample-every 30 --yolo

```

Отчёт `result.json` содержит метаданные прогона, покадровые ответы модели
(`command`, `reason`, `confidence`, `raw`), метрики каждого кадра (TTFT, TPS,
время) и агрегаты (средние, пропускная способность, распределение команд).

## Оценка качества вывода (без разметки)

```bash
python evaluate.py result.json
python evaluate.py result.json --output eval.json
```

Считает по одному `result.json`, без эталонных меток: валидность формата,
распределение и энтропию команд (вырожденность), временну́ю стабильность
(залипание/дёрганье), разнообразие и согласованность `reason`, статистику
`confidence`, сводку производительности и итоговые флаги-предупреждения.
